This is the most common, bread-and-butter header/navbar. Since the site is only 600px wide, I want another approach for the navigation. Beside that, I'm also a bit bored with this basic navigation style.

## "Essentialist" style

[Maxime's](https://maximevaillancourt.com/) website is peak essentialism. Focus only on what's _necessary_ and remove everything that doesn't contribute to the main purpose. Which in this case, blogging.

Sadly, I can't implement his style (_I've tried..._) for this site since I have categorization. I have [writings](/writings) and [projects](/projects) page, which is completely different. I have a non-blog page like [now](/now), [about](/about), and [guestbook](/guestbook) which I can't display alongside blog pages. I have to put it in a different place, a menu or navigation.

This website is unique, it's _headerless_. Once you visit [Manu's](https://manuelmoreale.com/) home page, you will be greeted by his latest blog post content. After reading the post, behold! There are navigation links underneath it, right after "**What should you do next?**". It's wrapped in a paragraph, descriptive style, which is a cool approach.

## Card component

[Manu's](https://manuelmoreale.com/) navigation style reminds me when I'm working with Table Of Contents in Microsoft Word. You have a heading title with a page number in the right-most side, with dots in the middle that connect and guide the viewers eye.
